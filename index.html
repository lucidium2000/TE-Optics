⌖ TE Optics2.12

Bulk-create ThousandEyes tests using your existing browser session. No API tokens needed — this tool piggybacks on the session you already have open in app.thousandeyes.com.

1

Log into ThousandEyes

Open app.thousandeyes.com in your browser and make sure you’re signed in.

2

Launch the panel

Bookmarklet (Chrome/Edge)
Drag this button to your bookmarks bar, then click it while on app.thousandeyes.com:

TE Optics Drag & drop onto your bookmarks toolbar. If hidden, press Ctrl+Shift+B (Chrome/Edge) to show it.

Updating? Remove your old TE Optics bookmark from the bar, then drag the button above again to add a replacement. Bookmarklets store a copy of the script from when the bookmark was saved, so replacing the bookmark is how you get the latest version.

3

Use the panel

After you launch TE Optics from your bookmark, the same sidebar shows different tabs depending on where you are in app.thousandeyes.com: Dashboards (the /dashboard experience) vs the rest of the app for enterprise agent tests.

Dashboards — On a dashboard URL you get Backup and Restore: import or refresh dashboard JSON from the app, save it to disk, upload a backup back to ThousandEyes, optionally remove all widget filters and clear virtual-agent fields on restore, load the account-group dashboard list for cleanup (sort and bulk delete), plus advanced diagnostics for /namespace/dash-api and related traffic.

Enterprise tests — On other pages the panel opens to Manage tests; use Create test to expand bulk create. the panel verifies your session, loads agents for the current account group, and supports bulk create and bulk edits. Supported types for creation:

  • HTTP Server — monitor web endpoints
  • Agent-to-Server — network connectivity
  • Page Load — full page render timing

Enter multiple targets (one per line) when bulk-creating tests.

How it works: The script runs directly inside the app.thousandeyes.com page and automatically detects your active session. It tries cookie auth first, then searches browser storage for tokens, and finally intercepts live TE app requests to capture the auth header — no separate server or manual token entry required.
